Data v4.0.0

Overview#

Most data methods are scoped to entry types. All Data endpoints are addressed {baseurl}/api/rest/v4/data/

Cells - These endpoints returns data as Key-Value pairs. These endpoints works primarily with Ids. When specifying entries and fields they expect the Ids of those items. These endpoints supports the full feature set of the API.

NameTypeEndPointDescription
EntriesGET /entrydata/{entryTypeId}/entriesGets all entries - only Primary Key and Entry Id
HistoryGET /entrydata/{entryTypeId}/entries/historyReturns modified and deleted entries from DateTime
FilterPOST /entrydata/{entryTypeId}/filtersReturns Ids of filtered entries
GetPOST /entrydata/getReturns all meta-data for entries except for binary data
Historical GetPOST /entrydata/getHistoricalDataReturns entry data at DateTime specified
CreatePOST /entrydata/{entryTypeId}Creates new entries
UpdatePUT /entrydata/{entryTypeId}Updates existing entries
DeleteDELETE /entrydata/{entryTypeId}Deletes entries
FilesGET /entryfiles/{entryId}/fields/{fieldId}Returns binary data

Rows - These endpoints supports most of the capabilities of the API. They can work with either Ids or string identifier for entryType and fields. These endpoints do not support the following features: Binary Data, Historical Data, Deleted Entries, Deletion, Delta Queries* (resolution is to the nearest day, if you wish to get more granular you'll need to use the Cells History endpoint)

NameTypeEndPointDescription
GetGET /entrydata/rows/{entryTypeId}Returns entries and meta-data. Supports Queries
CreatePOST /entrydata/rows/{entryTypeId}Creates new entries
UpdatePATCH /entrydata/rows/{entryTypeId}Updates existing entries

History#

Returns entries modified or deleted since the DataTime value provided. Accepts valid iso8601 DateTime formats

Request#

GET {{baseUrl}}/api/rest/v4/data/entrydata/{{entryTypeId}}/entries/history
?modifiedSince={{$datetime iso8601 -6 M}}
Authorization: {{auth}}
